OWASP Broken Web Applications

This is the VM for the Open Web Application Security Project (OWASP) Broken Web Applications project. It contains many, very vulnerable web applications. More information about this project can be found at http://www.owaspbwa.org/.

!!! This VM has many serious security issues, we strongly recommend that you run
it only in "host only" or "NAT" network in the virtual machine settings !!!

The following vulnerable web applications are running on the VM (listed in no particular order).

Intentionally Vulnerable Applications:

  • OWASP WebGoat version 5.3.x (Java, use username=guest, password=guest, home page)
  • OWASP Vicnum version 1.4 (PHP/Perl, home page)
  • Mutillidae version 1.5 (PHP, home page)
  • Damn Vulnerable Web Application version 1.07x (PHP, use username=admin, password=admin, home page)
  • Ghost (PHP, home page)
  • Peruggia version 1.2 (PHP, home page)
  • OWASP CSRFGuard Test Application version 2.2 (Java, home page)
  • Mandiant Struts Forms (Java/Struts)
  • Simple ASP.NET Forms (ASP.NET/C#)
  • Simple Form with DOM Cross Site Scripting (HTML/JavaScript)

Old Versions of Real Applications:

  • WordPress version 2.0.0 (PHP, released December 31, 2005, home page)
  • phpBB version 2.0.0 (PHP, released April 4, 2002, home page) - CURRENTLY NOT WORKING
  • OrangeHRM version 2.4.2 (PHP, released May 7, 2009, home page)
  • GetBoo version 1.04 (PHP, released April 7, 2008, home page)
  • GTD-PHP version 0.7 (PHP, released September 30, 2006, home page)
  • Yazd version 1.0 (Java, released February 20, 2002, home page)

OWASP Demonstration Applications (no known vulnerabilities):

  • OWASP AppSensor Demo Application (Java, home page)
For more information about the specific versions of applications running and how to adminsiter this VM, see http://code.google.com/p/owaspbwa/wiki/ProjectSummary.

For usernames to use with these applications, see http://code.google.com/p/owaspbwa/wiki/ApplicationAccounts.

For information about the known vulnerabilities in these applications (or to submit some), visit http://sourceforge.net/apps/trac/owaspbwa/report/1.
